如何优化你的安卓应用：电量篇（一）了解电量消耗

发表于 2017-04-09 | 分类于 Android

安卓手机的耗电问题一直饱受诟病，各个手机厂商也是一直各种在电池容量上搞噱头竞争。但用过安卓手机的人都有一个感受：一开始待机挺长时间的，后来用着用着就有变成一天一冲、两冲了。原因很多，一方面是手机的硬件老化；另一方面也是当前安卓应用生态的问题了：开发者永远只希望用户能多用自己的APP，能多推送自己的信息给用户。

另外，你也会有一个感受：假如你外出，你会特意把网络什么的关掉，这样你的手机一天下来居然还剩了很多电！！曾经我的用了2年多的小米1手机关掉后当成功能机使用居然待机2-3天没问题！！

手机的电量消耗

作为一个开发者，直观感受上电量的消耗无非就是几点：手机硬件（比如屏幕、声音）、传感器（GPS、重力感应等）、网络、应用CPU消耗这几大块。但是手机又是如何统计你的App的电量消耗的呢？是否你应用打开时屏幕的消耗都算你的APP的头上呢？等等一堆细节问题就会带来了。本文就讨论这些细节，不过还是先从外层一层一层剥开电量的面纱。

开机使用中的电量消耗

对于开机使用中，手机的最大消耗就在于硬件了，比如CPU、网络和屏幕。用户能做的只能是通过选择较好的网络环境（比如wifi）、调节屏幕亮度以及关闭其他不用的APP和硬件服务来提高使用时长、少玩一些需要高运算的游戏等。国内的自定制rom也在这方面对APP做了很多约束，比如后台挂起应用等，这也给应用开发者带来了一堆头痛的问题（比如后台保活）。要怪就怪谷歌不按套路出牌退出中国市场吧… …（提供了一堆的服务中国用户却无法享受，生态一直难以维护）。

网络环境与耗电

不同的网络环境电量的消耗也是不同的，作为一个开发者，了解不同的网络环境对电量的影响还是很有必要的，这里面我就没有专门再做测试了，就借用网上文章来说明下：

（1）在待机情况下，CPU以及网络模块完全处于休眠状态，因此也是最省电。

（2）灭屏情况下传输，CPU被激活，耗电显著增加，即便是处理1K的心跳包，电量消耗也会是待机的6倍左右（45mA）；

（3）在灭屏情况下，高负载的download时候，不同网络下电量消耗也明显不同，其中WiFi最省电（70mA），3G（270mA）和2G（280mA）相当，是WiFi的4倍左右。

综上，不同网络环境下耗电还是有差距的，APP开发需要做的就是根据不同的网络环境做相应策略的调整来优化APP的性能，比如尽可能的在wifi环境下做耗费网络的操作。

PowerProfile

该类通过从XML文件中读取值，这些值代表了系统各个硬件部件在不同状态时的电量消耗值，比如CPU在空闲时、活动时等。XML为：R.xml.power_profile，值以Map的形式存储。

上面这个接口可以看出这些部件的耗电量也会分等级的，比如CPU运行在400MHZ和1GHZ的情况下单位时间内耗电量也是不同的。

Radio（无线电网络）功耗

初始化：

主要从profile中提取radio在active、scan的时候平均电流，以及在不同信号强度下的电流。

计算：

这个统计也比较简单：

  1. 优先以无线网络active时长为基准，active time * powerRadioOn。
  2. 假如active time为0，则根据网络收发包的数量为基础计算，(Rx + Tx) * anvPerPacketPower。

可见，就算应该考虑集中处理网络操作，减少active time才是最有效的节省电量消耗的方法，packet数量次之。

总结

电量消耗算法总的来说在CPU、Radio、Wakelock上会复杂点，也是平时最容易忽略掉的。其他基本属于硬件类，业务属性比较清晰也是最容易解决的。

对于CPU应该尽可能的让其在低负载情况下运行，因为高负载后单位时间电流也会不止翻倍的增加。其次就是尽可能减少移动网络的active time。

Last

本文的代码均以6.0(v23)版本做分析。安卓上电量统计的大致描述可如下：

  1. 应用以及系统运行时记录时长等数据到batterystats.bin中。
  2. BatteryStatsImpl提供了从batterystats.bin提取各种统计数据的能力。
  3. PowerProfile提供了手机各个部件的单位电流量数值，数值记录在power_profile.xml中。
  4. BatteryStatsService做为服务提供数据给系统和应用。
  5. BatteryStatsHelper则从BatteryStatsImpl中获取统计数据，然后再根据PowerProfile计算出最终的电量消耗。
